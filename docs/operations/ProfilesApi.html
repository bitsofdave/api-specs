This is the API

profilesGet

This method returns an array of `profiles`, whose format and content will be the same as the show profile endpoint. This endpoint uses pagination through two optional parameters. The first, limit, defaults to 30 and accepts EITHER an integer value, OR the string 'all' to return all records. The second, after, should be the id of the first record BEFORE the recrods that you want to retrieve.

URL

https://company.namely.com/api/{version}/profiles

HTTP Method

GET

Response Type

inline_response_200

Parameters

  • Query: per_page Integer

    Integer limit of records retrieved.

  • Query: page Integer

    The page number to retrieve.

  • Query: sort String

    An attribute to sort by fields on profiles. Allowed attributes are first_name, last_name, and job_title. Prepend with a - (minus) sign to reverse the order.

  • Query: filter Map(String)

    A hash containing a filter you'd like to apply before profiles are returned.

profilesPost

The fields that you can set and will receive are a combination of Namely's defaults fields and custom fields that exist only for your company.

URL

https://company.namely.com/api/{version}/profiles

HTTP Method

POST

Response Type

inline_response_200

Parameters

  • Query: first_name String

    First name

  • Query: last_name String

    Last name

  • Query: email String

    Email.

  • Query: user_status String

    User status, should be \"active\" or \"inactive\"

  • Query: start_date String

    Start date

  • Query: departure_date String

    Depature date

  • Query: gender String

    Gender, should be \"male\", \"female\"

  • Query: marital_status String

    Marital status, should be \"single\", \"married\", \"civil partnership\", \"separated\", or \"divorced\"

  • Query: job_title String

    Valid job title GUID

  • Query: resume String

    Resume file GUID

  • Query: image String

    Image file GUID

  • Query: office String

    Office address, hash[see below]

  • Query: home String

    Home address, hash[see below]

profilesidGet

This method allows users to retrieve a `profile`. The profile entry will contain additional information from all fields that your company has (default and custom) that are currently supported by the API and that the current user has access to for the given user. You can get a full list of accessible fields, as well as information on formatting them from the fields API endpoint.

URL

https://company.namely.com/api/{version}/profiles/:id

HTTP Method

GET

Response Type

inline_response_200_1

Parameters

  • Query: id String
